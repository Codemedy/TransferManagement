# Simple Payment Service

Данный проект представляет собой реализацию сервиса для управления платежными операциями. Сервис позволяет создавать платежи, изменять их состояние и просматривать историю транзакций по клиентам.

## 🛠 Технологический стек
* **Java 21**: Использование современных фич языка.
* **Spring Boot 4.0.5**: Основа приложения.
* **Spring Data JPA**: Работа с базой данных.
* **PostgreSQL**: Реляционная БД для хранения данных.
* **Gradle**: Система сборки проекта.
* **Docker & Docker Compose**: Контейнеризация приложения и базы данных.
* **SpringDoc OpenAPI (Swagger)**: Автоматическая документация API.

## 🚀 Функциональные возможности
В рамках задания реализованы следующие механизмы:
1.  **Создание платежа**: Регистрация новой транзакции со статусом `PENDING` по умолчанию.
2.  **Валидация данных**: Проверка суммы на положительное значение и ограничение списка допустимых валют (`KZT`, `USD`, `EUR`, `RUB`).
3.  **Управление статусами**: Возможность подтверждения (`CONFIRMED`) или отмены (`CANCELED`) платежа. Реализована бизнес-логика, запрещающая изменять статус уже завершенных операций.
4.  **История клиента**: Получение списка всех транзакций по конкретному `clientId`.
5.  **Обработка исключений**: Глобальный обработчик ошибок возвращает корректные HTTP-статусы:
    * `404 Not Found`: если платеж или клиент не существуют.
    * `400 Bad Request`: при нарушении бизнес-логики или ошибках валидации.

## 📦 Запуск проекта

### 1. Подготовка окружения
Убедитесь, что у вас установлены **Docker** и **Docker Compose**.
В корне проекта должен находиться файл `secret.env` с настройками базы данных, это было сделано, для того чтобы скрыть данные:
```env
DB_NAME=payment_db
DB_USERNAME=postgres
DB_PASSWORD=postgres
```

### 2. Сборка и запуск
Выполните команду в терминале:
```bash
docker compose up -d --build
```
Эта команда автоматически соберет JAR-файл внутри Docker-контейнера (используя многоэтапную сборку), поднимет базу данных PostgreSQL.

После поднятия контейнера базы данных PostgreSQL можете запустить приложение по команде:
```bash
.\gradlew bootRun
```

## 📖 Документация API (Swagger)
После успешного запуска документация доступна по адресу:
👉 **[http://localhost:8080/swagger-ui/index.html](http://localhost:8080/swagger-ui/index.html)**

В интерфейсе Swagger вы можете интерактивно протестировать каждый эндпоинт:
* `POST /payments`: Создание.
* `GET /payments/{id}`: Проверка статуса.
* `POST /payments/{id}/confirm`: Подтверждение.
* `POST /payments/{id}/cancel`: Отмена.
* `GET /clients/{clientId}/payments`: Поиск по клиенту.
